# 기여 가이드 (Contributing Guide)

Universal Notion Agent 프로젝트에 기여해 주셔서 감사합니다! 🎉

## 🤝 기여 방법

### 1. 이슈 생성

버그 발견이나 기능 제안이 있으시면 먼저 이슈를 생성해주세요:

- **버그 리포트**: 발생한 문제, 재현 방법, 예상 동작을 명확히 기술
- **기능 제안**: 새로운 기능의 필요성과 구체적인 사용 사례 설명

### 2. Fork 및 브랜치 생성

```bash
# 1. Repository Fork
# 2. 로컬에 클론
git clone https://github.com/YOUR_USERNAME/Universal_Notion_Agent.git
cd Universal_Notion_Agent

# 3. 기능 브랜치 생성
git checkout -b feature/amazing-feature
```

### 3. 개발

**브랜치 네이밍 컨벤션**:

- `feature/`: 새로운 기능 추가
- `fix/`: 버그 수정
- `docs/`: 문서 개선
- `refactor/`: 코드 리팩토링

**커밋 메시지 컨벤션**:

```
타입: 간단한 설명

상세한 설명 (선택 사항)

관련 이슈: #123
```

**타입**:

- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 변경
- `style`: 코드 포맷팅
- `refactor`: 코드 리팩토링
- `test`: 테스트 추가/수정
- `chore`: 기타 변경사항

### 4. Pull Request

```bash
# 1. 변경사항 커밋
git add .
git commit -m "feat: Add amazing feature"

# 2. 원격 저장소에 푸시
git push origin feature/amazing-feature

# 3. GitHub에서 Pull Request 생성
```

**PR 설명에 포함할 내용**:

- 변경사항 요약
- 관련 이슈 번호
- 테스트 방법
- 스크린샷 (UI 변경 시)

## 📋 개발 가이드라인

### 문서 작성 기준

1. **명확성**: 누구나 이해할 수 있도록 명확하게 작성
2. **일관성**: 기존 문서의 스타일과 구조 유지
3. **예시**: 구체적인 사용 예시 포함
4. **한국어**: 한국어로 작성 (영어 기술 용어는 병기)

### 코드 패턴 기준

1. **재사용성**: 범용적으로 사용 가능한 패턴
2. **플랫폼 독립성**: 특정 플랫폼에 종속되지 않도록
3. **에러 처리**: 명확한 에러 메시지와 해결 방법 제시
4. **주석**: 복잡한 로직에는 주석 추가

### 테스트

새로운 기능이나 버그 수정 시:

1. 최소 2개 이상의 플랫폼에서 테스트
2. 에러 케이스 확인
3. 대용량 파일 처리 확인

## 🎯 기여 영역

### 우선순위가 높은 기여

- **버그 수정**: 이슈로 보고된 버그 수정
- **문서 개선**: 오타 수정, 설명 보완, 예시 추가
- **에러 처리**: 새로운 에러 케이스 추가 및 해결 방법 문서화
- **플랫폼 지원**: 새로운 MCP 플랫폼 지원 추가

### 환영하는 기여

- **번역**: 영어 버전 문서 작성
- **예시**: 다양한 사용 사례 추가
- **최적화**: 성능 개선 및 코드 최적화
- **테스트**: 자동화 테스트 추가

## 📝 커뮤니티 규칙

### 행동 강령

- 존중: 모든 기여자를 존중합니다
- 건설적: 비판은 건설적으로, 대안을 제시합니다
- 협력적: 함께 더 나은 프로젝트를 만듭니다

### 질문 및 토론

- **이슈**: 버그, 기능 제안
- **Discussions**: 일반적인 질문, 아이디어 공유
- **PR 리뷰**: 코드 개선 제안
